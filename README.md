# buti

A minimal **Java + Maven** starter project (generated from the Maven quickstart archetype) with a simple application class, unit test scaffold, and Jenkins pipeline.

## What this project does

- Builds a Java JAR with Maven.
- Runs a baseline unit test suite with JUnit.
- Provides a CI pipeline (`Jenkinsfile`) with checkout, build, test, and deploy placeholder stages.

Current sample behavior:

```text
Hello World!
```

---

## Tech stack

- **Java**
- **Maven** (`pom.xml`)
- **JUnit 3** (legacy; can be upgraded)
- **Jenkins Declarative Pipeline**

---

## Project layout

```text
.
├── pom.xml
├── Jenkinsfile
├── src/
│   ├── main/java/com/example/App.java
│   └── test/java/com/example/AppTest.java
└── README.md
```

> `target/` is generated by Maven builds and should be treated as build output.

## Prerequisites

- JDK 8+ (JDK 11+ recommended)
- Maven 3.6+

Check your setup:

```bash
java -version
mvn -version
```

---

## Build and test

From repo root:

```bash
mvn clean package
```

Run tests:

```bash
mvn test
```

---

## Run the app

### From compiled classes

```bash
mvn clean compile
java -cp target/classes com.example.App
```

### From packaged artifact

```bash
mvn clean package
java -cp target/maven-example-1.0-SNAPSHOT.jar com.example.App
```

> Note: the current POM does not configure an executable `Main-Class` manifest, so `java -jar ...` is not enabled yet.

---

## CI (Jenkins)

The `Jenkinsfile` defines these stages:

1. **Checkout**
2. **Build** (`mvn clean package`)
3. **Test** (`mvn test`)
4. **Deploy** (placeholder)

This is a good base for adding artifact publishing and environment-specific deployment logic.
